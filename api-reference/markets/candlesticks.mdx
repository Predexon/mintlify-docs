## Overview

Fetch historical candlestick (OHLCV) data for charting and analysis. Data is normalized to the YES side perspective - NO prices are converted as (1 - price).

<ParamField path="condition_id" type="string" required>
  The condition ID for the market
</ParamField>

<ParamField query="start_time" type="integer">
  Unix timestamp (seconds) for start of time range. Defaults to earliest available data.
</ParamField>

<ParamField query="end_time" type="integer">
  Unix timestamp (seconds) for end of time range. Defaults to latest available data.
</ParamField>

<ParamField query="interval" type="integer" default="1">
  Candlestick interval in minutes:
  - `1` - 1 minute (max 1 week range)
  - `60` - 1 hour (max 1 month range)
  - `1440` - 1 day (max 1 year range)
</ParamField>

<ResponseField name="condition_id" type="string">
  The market condition ID
</ResponseField>

<ResponseField name="candlesticks" type="array">
  Array of candlestick data points
  <Expandable title="Candlestick structure">
    <ResponseField name="end_period_ts" type="integer">
      Unix timestamp for end of this candle period
    </ResponseField>
    <ResponseField name="price" type="object">
      <Expandable>
        <ResponseField name="open" type="number">Opening price (0-1)</ResponseField>
        <ResponseField name="high" type="number">Highest price</ResponseField>
        <ResponseField name="low" type="number">Lowest price</ResponseField>
        <ResponseField name="close" type="number">Closing price</ResponseField>
        <ResponseField name="open_dollars" type="string">Opening price formatted as dollars</ResponseField>
        <ResponseField name="high_dollars" type="string">Highest price formatted as dollars</ResponseField>
        <ResponseField name="low_dollars" type="string">Lowest price formatted as dollars</ResponseField>
        <ResponseField name="close_dollars" type="string">Closing price formatted as dollars</ResponseField>
        <ResponseField name="mean" type="number">Mean price for period</ResponseField>
        <ResponseField name="mean_dollars" type="string">Mean price formatted as dollars</ResponseField>
        <ResponseField name="previous" type="number">Previous candle's close</ResponseField>
        <ResponseField name="previous_dollars" type="string">Previous close formatted as dollars</ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="volume" type="number">
      USD notional volume for this period
    </ResponseField>
    <ResponseField name="trades_count" type="integer">
      Number of trades in this period
    </ResponseField>
  </Expandable>
</ResponseField>

```python Python
import requests

response = requests.get(
    f"https://api.predexon.com/v1/polymarket/candlesticks/{condition_id}",
    headers={"x-api-key": "YOUR_API_KEY"},
    params={
        "interval": 60,
        "start_time": 1704067200,
        "end_time": 1704153600
    }
)
candlesticks = response.json()["candlesticks"]
```

```typescript TypeScript
const response = await fetch(
  `https://api.predexon.com/v1/polymarket/candlesticks/${conditionId}?interval=60&start_time=1704067200&end_time=1704153600`,
  { headers: { "x-api-key": "YOUR_API_KEY" } }
```

## Usage Notes

<AccordionGroup>
  <Accordion title="Time range limits">
    To ensure reasonable response sizes, each interval has a maximum time range:
    - 1-minute: 1 week
    - 1-hour: 1 month
    - 1-day: 1 year
  </Accordion>
  <Accordion title="Price normalization">
    All prices are normalized to the YES perspective. This means if you're tracking the NO side, the displayed prices show (1 - actual_no_price).
  </Accordion>
  <Accordion title="Volume aggregation">
    Volume is the combined USD notional from both YES and NO token trades within each period.
  </Accordion>
</AccordionGroup>
